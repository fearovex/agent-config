- **Tasks 2.1, 2.2, 3.1, 3.2 are independent of each other** — they modify four different skill files with no cross-dependency; they can be applied in any order after Phase 1 is complete.
- **Tasks 4.1, 4.2, 4.3 all modify the same file** (`CLAUDE.md`) — apply them sequentially in a single editing pass to avoid conflicts.
- **Task 4.4 modifies `ai-context/architecture.md`** — append the new row inside the existing artifact table, before the `## Key architectural decisions` section; do not modify the `[auto-updated]` sections.
- **Task 5.1 is a read-only verification** — if `install.sh` already handles `ai-context/` recursively, no file change is needed. The task is complete when the implementer confirms this in writing.
- **Domain slug matching heuristic (used in tasks 2.1 and 2.2)** — as specified in design.md: split change slug on hyphens → discard single-char stems → for each feature file: match if `domain` appears in change-name OR any stem appears in domain → return all matches. `_template.md` and files with a leading underscore are always excluded from matching.
- **`openspec/config.yaml` `feature_docs:` block** — per the proposal's explicit exclusion, do NOT uncomment or activate the `feature_docs:` block. It remains commented out in V1.
