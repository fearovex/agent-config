## Implementation Notes

- The skill is **purely declarative** (Markdown + embedded prompts in SKILL.md). No Bash scripts, no Node.js helpers, no external tool invocations. Claude performs all transformation in-context when the skill is executed.
- The transformation prompts in Phases 3.1–3.3 are the critical quality gate. They must be precise about what to strip and what to retain. Reference the strip table in `specs/config-export-targets/spec.md` and the data-flow diagram in `design.md` when writing the prompts.
- The skill must NOT write any files to `skills/config-export/` beyond `SKILL.md` — no helper files, no templates, no sub-scripts.
- Tasks in Phase 2 and Phase 4 add sections to the same file (`skills/config-export/SKILL.md`). The implementer must follow the order in `## Process` (Step 1 → 2 → 3 → 4 → 5) to keep the skill coherent for a reader.
- The generated-file banner format is fixed (see `design.md` Interfaces > Generated file header). Use it verbatim in all three transformation prompts.
